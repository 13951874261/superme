# STEP 2 选择材料 功能测试案例说明文档

本篇文档定义了“STEP 2 选择材料”中**本地文档**、**网页提取**和**视频字幕**三个核心子功能的测试案例。测试数据和文件已预置在项目根目录的 `test-materials` 目录中。

---

## 1. 测试用例 1：本地文档上传与提纯 (PDF/Word/TXT/MD)

### 1.1 MD 格式测试
*   **对应需求**：支持 Markdown 格式本地文档选择、上传并执行 Dify 向量化和提纯入库。
*   **菜单路径**：主页 -> `STEP 2 选择材料` -> 选择 `本地文档` 选项卡 -> 点击 `选择本地文档` 按钮
*   **测试数据**：`test-materials/test_doc.md` (已预置，包含 Acme 与 Beta 商务谈判的英文语料和词汇表)
*   **测试步骤**：
    1. 在文件选择器中选择 `test-materials/test_doc.md`。
    2. 验证前端是否显示已选择的文件名 `test_doc.md`。
    3. 点击下方右侧 `开始上传并提纯` 按钮。
    4. 观察右侧 `STEP 3 执行与预览` 区域的执行日志栏。
*   **预期结果**：
    1. 页面正确加载文件名 `test_doc.md`，状态变为就绪。
    2. 点击执行后，状态显示 `后端正在执行...`，日志栏逐步打印：“清空 Dify 知识库旧文档...”、“正在上传新文件至 Dify 知识库...”、“正在轮询嵌入状态...”、“卡片生成一键提纯流程启动...”。
    3. 提纯完成后，日志最后显示 `全部处理完成`，生词本等词汇卡片能够接收更新并渲染出文档中的相关专业词汇（如 widget, procurement, premium, logistics）。

## 2. 测试用例 2：网页提取测试 (URL Fetch)

### 2.1 合法公共网页提取
*   **对应需求**：支持输入合法公共网页 URL 进行正文抓取、转换为 Markdown 预览并载入提纯。
*   **菜单路径**：主页 -> `STEP 2 选择材料` -> 选择 `网页提取` 选项卡
*   **测试数据**：`https://en.wikipedia.org/wiki/Business_negotiation` (维基百科公共页面)
*   **测试步骤**：
    1. 在输入框输入上述公共 URL。
    2. 点击 `预览` 按钮。
    3. 观察网页提取的加载状态和渲染的预览区域。
    4. 确认无误后，点击 `确认提取`。
    5. 点击 `开始上传并提纯` 启动一键提纯。
*   **预期结果**：
    1. 点击 `预览` 后展示“正在提取...”加载动画，随后在行内（或全屏弹窗）成功显示抓取的网页标题 `Business negotiation` 和正文的 Markdown 格式预览。
    2. 点击 `确认提取` 后，输入栏重置，主界面加载名为 `Business negotiation.md` 的虚拟文件。
    3. 点击 `开始上传并提纯`，将抓取的网页 Markdown 数据发送给后端提纯，知识库完成更新。

### 2.2 非法/私有 IP URL SSRF 拦截测试
*   **对应需求**：系统安全防御，对于 localhost/127.0.0.1 或内网私有 IP 的 URL 请求进行拦截，防止 SSRF 漏洞。
*   **菜单路径**：同上。
*   **测试数据**：`http://localhost:3001/api/materials/fetch-url` 或 `http://192.168.1.1`
*   **测试步骤**：
    1. 在输入框输入内网或回环地址。
    2. 点击 `预览` 按钮。
*   **预期结果**：
    1. 系统报错并拦截，前端弹出红色提示框，错误内容类似：“网页抓取失败：目标 URL 格式非法或属于受限的网络地址”。
    2. 后端没有发出实质性的内网请求，安全机制起效。

## 3. 测试用例 3：视频/音频转写测试

### 3.1 本地视频文件上传与转写
*   **对应需求**：支持拖拽或选择本地音视频文件，利用后端 FFmpeg 提取 16kHz MP3，上传至 Dify 语音识别并提纯。
*   **菜单路径**：主页 -> `STEP 2 选择材料` -> 选择 `视频字幕` 选项卡
*   **测试数据**：`test-materials/test_video.mp4` (已生成，内置测试英语音频轨)
*   **测试步骤**：
    1. 点击“拖入或选择本地视频文件”区域，选择 `test-materials/test_video.mp4`。
    2. 在“识别语言”下拉菜单中选择 `自动识别` (或指定 English)。
    3. 点击下方右侧 `开始上传并提纯` 按钮。
*   **预期结果**：
    1. 页面加载视频并准备就绪，显示对应的视频预览。
    2. 点击执行后，右侧日志栏输出：“启动 FFmpeg 音轨提取组件...”、“音轨提取成功 (16kHz 单声道 MP3)”、“正在上传至 Dify 语音转写平台...”、“正在提交 Dify 语音转写工作流...”。
    3. 成功获取转写的文字（即 transcript），自动生成虚拟材料文件 `${文件名}_transcript.md`，执行清空并导入 Dify 知识库流程。
