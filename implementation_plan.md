# 结合 Dify 的高阶词库与记忆系统完整实现方案

本方案旨在通过 Dify 的 LLM 工作流（Workflow）能力，彻底替换掉传统的三方词典 API，将“工具检索区”和“记忆生词本”打造成纯粹且高度定制的“AI 个性化职场辅助利器”。

## 架构概览

整个链路以**“前端（本地存储 + 调度） + Dify（动态引擎 + 数据提取）”**的形态协同工作：
1. **Dify 端角色**：充当“超级大脑”与“词典解析层”。
2. **前端端角色**：充当“交互展现层”、“艾宾浩斯统筹中枢”及“离线存储层”。

---

## 产品与功能细化（需求定义）

### 1. 工具区：三类词典（均由 Dify Workflow 生成结构化结果）

| 能力 | 定位 | 用户价值 | 建议 `type` 枚举值 |
|------|------|----------|-------------------|
| **现代汉语词典（写作用）** | 规范汉语书面表达 | 释义、词性、常用搭配、近义/易混辨析、得体用法；偏「怎么写对、写自然」 | `zh_modern` |
| **商务英英词典** | 职场英文在商务语境中的含义 | **英英释义** + 行业/商务语用说明 + **带场景标签的例句**（邮件、会议、谈判等） | `en_en_business` |
| **英汉词典（双向）** | 中英互译 + 职场例句 | **双向**：英→中 / 中→英（自动或显式指定）；**音标**（IPA 或通用体系）；**商务例句**（中英对照 + 场景） | `en_zh_bidirectional` |

**请求侧建议字段（与 Dify 开始节点变量一致）：**

- **必填**：`word`（词或短语）、`type`（上表枚举）。
- **可选**：`direction`（`auto` | `en_to_zh` | `zh_to_en`，仅 `en_zh_bidirectional` 使用；默认 `auto` 由模型根据 `word` 判断）、`user_context`（用户当前写作/业务一句话，用于例句贴场景）、`locale`（如 `zh-CN`）。

### 2. 生词本 + 艾宾浩斯复习

| 子需求 | 说明 | Dify / 前端分工 |
|--------|------|-----------------|
| **自动保存** | 保存练习中的：**词目**、本次返回的**结构化释义片段**、**例句**、**场景标签**、**词典类型 `type`**、**时间戳**；支持一键入库与「查完自动问是否加入」 | **持久化、去重、列表查询**：**前端**（IndexedDB 等）。Dify **不做**高频词汇库读写。 |
| **复习推送** | 按记忆曲线在「今日待复习」中呈现（角标、列表）；可选浏览器 **Notification**（需权限） | **调度与间隔计算**：**前端本地**（`nextReviewAt`、`level`、间隔策略）。 |
| **复习交互** | 卡片正反面、自评（忘记/模糊/记得）→ 更新 `level` 与 `nextReviewAt` | **纯前端**；保证弱网/离线仍可复习。 |

**艾宾浩斯参数（与下文章节一致，可做成可配置项）：** 例：`12h → 1d → 2d → 4d → 7d → 15d`；答错可降级或重置 `level`。可选后续迭代 **SM-2** 变体。

**可选 Dify 扩展（非必选）：** 复习会话中如需「根据旧卡再生成一道微测验」，可单独做一个短 Workflow；**主复习闭环不依赖 Dify**，避免成本与延迟。

## 核心实现设计

在 Dify 后台中需要建立 **Workflow 应用（工作流应用）**，通过 **API** 暴露给前端（如 `POST /v1/workflows/run`，以当前 Dify 版本文档为准），使用 **Workflow API Key**；**不要用对话型 Chat 应用**作为词典工具入口，否则 JSON 输出不稳定、难以填入固定 UI 晶格。

### 阶段 1：Dify 智能词典构建 (Dify Workflow) — 技术规格摘要

**目标**：单次调用返回 **单一、可 `JSON.parse` 的结构**（建议结束节点输出变量即为 JSON 文本，或由代码节点校验后输出）。

1. **输入**：见上文「请求侧建议字段」；开始节点变量名与前端请求体对齐。
2. **分支**：推荐 **单工作流** + `IF/else` 或「问题分类器」按 `type` 路由到 **3 个 LLM 节点**（各带独立 System Prompt），末尾用 **代码节点** 统一包装为 `{"ok": true, "type": "...", "payload": { ... }}`；解析失败时返回 `ok: false` 与 `error_code`。
3. **各类型最小 JSON 字段（须在 Prompt 中要求「仅输出 JSON、无 Markdown 代码围栏」）**：

| `type` | `payload` 核心字段 |
|--------|-------------------|
| `zh_modern` | `definition`, `usage_notes`, `collocations`, `confusable_pairs`, `example_sentences` 等 |
| `en_en_business` | `headword`, `pos`, `definitions_en`, `business_notes`, `scenarios[]`（含 `scene`, `example_en`） |
| `en_zh_bidirectional` | `direction_resolved`, `phonetic`, `translation_main`, `business_examples[]`（含 `zh`, `en`, `scene`） |

与旧版示例的对应：`hidden_meaning` 可并入商务类字段；`scenarios` 由「字符串数组」升级为「带场景标签的对象数组」，便于前端组件展示。

### 阶段 2：前端接入与检索 UI 开发 (Chat/Tool Area)
在 `Sidebar.tsx` 内开发轻量独立面板：
* **交互逻辑**：当点击左下角工具区的三大词典按钮时，呼出一个**专属检索小框**。
* **网络请求层**：通过与之前一致的 `fetch` API，向 Dify Workflow 的 `/workflows/run` 接口发送请求（需配置对应的 Workflow API Key）。
* **加载效果**：增加平滑的解析 Skeleton (骨架屏)，防止 Dify 大语言模型请求较慢带来的延迟感。
* **关键动作 - 加入记忆库**：解析完成后，结果顶部悬浮一个 `[一键加入艾宾浩斯记忆网]` 的按钮。

### 阶段 3：本地化艾宾浩斯记忆算法中枢 (Vocabulary Book)
（与上文「产品与功能细化 §2 生词本」一致。）Dify 不适合作为带时间戳的高频词汇库，采用**「前端数据中台化」**策略。

* **数据基座**：`IndexedDB` + Zustand 持久化（或等价方案）；**单机存储**即可满足 MVP。
* **生词本条目的建议字段**：`id`, `word`, `type`（`zh_modern` / `en_en_business` / `en_zh_bidirectional`）, `payload`（Dify 返回的 `payload` 快照或裁剪子集）, `examples`（例句数组，可与 `payload` 冗余一份便于卡片展示）, `tags`（场景标签）, `createdAt`, `level`, `nextReviewAt`, `lastReviewedAt`。
* **复习推演算法 (Algorithm)**：
  * `level` 与 `nextReviewAt` 由艾宾浩斯间隔驱动；答毕根据自评更新 `level` 并重算 `nextReviewAt`。
  * 间隔示例：12小时、1天、2天、4天、7天、15天（与下表一致时可配置）。
* **复习实训卡片 (Flashcard UI)**：
  * 点击“进入生词本”，覆盖当前右侧屏幕，呈现**深色沉浸式专注模式**。
  * 取出 `nextReviewTimeStamp < 当前时间` 的词汇。
  * 默认展示词汇，点击“空格”或手点屏幕翻转。根据回答（忘记、模糊、精准）更新它的复习衰减 `level` 并重算下一个时间戳。

> 以下实现细节需要您做出决策或提供您的意见，这将决定下一步的代码编写方向：

1. **API Keys 隔离要求**：由于 Dify 的“Workflow (工作流) 发起 URL” 和普通的 “Chat 发起 URL” 对应完全不同的密钥管理机制，我们需要独立再放置一个 Dify 工作流 Token。您目前是否能在 Dify 中建立一个**带有明确输入和输出变量的 Workflow（工作流）应用**？（如果暂时不能，我会教您如何在 Dify 页面上拖拽实现）。
2. **生词本入口布局**：进入“艾宾浩斯复习流”时，您是希望它在**左边侧边栏直接原地折叠展开**，还是点击后与刚刚的“全局对话大屏”一样，在**大显示区域铺开一层深色抽屉**执行复习？建议用大空间。
