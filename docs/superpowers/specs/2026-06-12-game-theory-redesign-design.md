# 驭心博弈（人性认知与高层博弈）模块重构设计方案

本规范详细定义了“驭心博弈”板块的功能升级、交互改良以及全局行政级 UI/UX 特效与声音的集成方案。

---

## 一、 整体 UI/UX 与多媒体规范

为了达到极简高端商务风格与企业级权威感，废弃原有的炫光与高饱和度色调，采用极致克制的行政美学。

### 1. 声音系统规范 (Sound)
完全复用并导入 `src/utils/soundEffects.ts` 中已定义的底层音效：
- **`playClick`**（极致极简水滴声）：用于点击按钮、切换选项、选中卡片、表单聚焦等。
- **`playPageTurn`**（真实沙沙纸张翻页声）：用于 Tab 选项卡切换、弹出新面板、成功保存人性原型。
- **`playGentleWarning`**（温柔平缓的低频和弦提示音）：用于评估分数过低（< 50分）或输入校验未通过时的退路预警。

### 2. 视觉光效规范 (Light)
- **色彩设计**：全面净化所有霓虹色、高饱和度色彩（如 Indigo-600/emerald-500 的渐变），统一采用以 Zinc/Slate 为核心的低调冷灰度调。
- **卡片美学**：卡片使用极浅的 `border border-zinc-200/80` 细线边框，搭配超柔和的微投影 `shadow-[0_4px_20px_-4px_rgba(9,9,11,0.04)]`。
- **状态过渡**：卡片与按钮被选中时，使用柔和的灰白过渡光效 `bg-zinc-50 border-zinc-400`。
- **严禁项目**：严禁在系统中使用霓虹渐变条、背景发光炫球或粒子背景。

### 3. 特效与动效规范 (Electricity)
- **极简纸屑 (Confetti)**：调用 `canvas-confetti` 库，在成功破局（打分 > 70分 / 满分100）或成功保存人性原型档案时，喷洒 Zinc-100、Zinc-200、Zinc-300 及纯白色组成的冷灰色调单色碎纸屑。
- **选项卡平滑移动**：Tab 页签使用 `framer-motion` 的 `layoutId="activeTabUnderline"` 实现选中状态下 Zinc-900 下划线平滑移动效果。
- **过渡动画**：四个模块的内容区切换采用 `motion.div` 的位移淡入动效（`initial={{ opacity: 0, x: 8 }} animate={{ opacity: 1, x: 0 }}`）。

---

## 二、 4 大核心训练模块重组交互方案

四大模块由顶层极简 Tab 导航控制。

```
+-------------------------------------------------------------------------+
|  [ 高管斗争案例库 ]   [ 驭人术与人性档案 ]   [ 博弈实操推演 ]   [ 顶层认知升维 ]  |
+-------------------------------------------------------------------------+
|                                                                         |
|                              ( 内容展示区域 )                            |
|                                                                         |
+-------------------------------------------------------------------------+
```

### 1. 模块一：真实高管斗争案例库（实战与研判）
- **左侧切换栏**：垂直排列“体制内权斗局”、“外企权斗局”、“以下克上战”三种环境切换按钮。点击时触发水滴声。
- **案例详情卡片**：展示选中环境下的预设案例，支持用户自行编辑或手动输入。
- **强制拆解表单**：要求用户必须完成以下四项研判：
  1. *利益结构分析*（`stakeholder_interests`）
  2. *善/恶动机透视*（`motives_analysis`）
  3. *对方权力弱点*（`weaknesses`）
  4. *博弈关键节点*（`key_points`）
- **研判提交**：
  - 点击“提交研判”按钮，触发水滴声与行政极简加载动画。
  - 将四项字段的内容拼接为 `"① 利益结构：... \n② 善/恶动机：... \n③ 权力弱点：... \n④ 博弈关键节点：..."`，传入 `GameTheoryAnalyzeInput.user_answer` 字段。
  - 调用 `runGameTheoryAnalysis` API。
  - 若打分高（>= 70分）则喷洒单色碎纸屑，播放纸张翻页声。
  - 报告展示：评估报告卡片采用微投影和灰白底色。

### 2. 模块二：驭人术与人性档案（高层成事术）
- **手段展示栏**：平铺展示“上级驭下手段”卡片（恩威并施、分而治之、制衡术、边缘化等）与“以下克上手段”卡片（借势上位、构建联盟、信息垄断、软对抗等）。点击弹出释义，播放水滴声。
- **人性原型档案库（录入与存档）**：
  - 用户可在表单中手动录入遇到人物的原型（姓名、类型如利益/恐惧/面子/安全感驱动型、特征及支配防范策略）。
  - 点击“录入档案册”调用 `upsertPersonalPrototype` 接口。
  - 成功后播放翻页声并喷洒 Confetti 纸屑，同时刷新右侧的原型库列表。
  - 原型卡片以微投影和冷灰质感排版呈现。

### 3. 模块三：博弈论实操推演（理论落地）
- **理论框架区**：在顶部展示“囚徒困境”、“智猪博弈”、“信息不对称”、“冷酷触发”等经典博弈理论的白话释义，支持平滑切换并伴随水滴声。
- **对战选择下拉菜单**：
  - 提供下拉菜单选择不同的虚拟对手与场景：
    1. *空降的改革派 VP* (博弈局势：利益冲突与制度架空)
    2. *任人唯亲的常务副总* (博弈局势：派系打压与同盟拉拢)
    3. *多疑的总监* (博弈局势：功劳争夺与信息屏蔽)
- **模拟推演输入**：
  - 展示所选场景与对手对你的刁难情境。
  - 用户输入你的反制策略。
  - 调用 `runGameTheoryAnalysis`，将选择的博弈模型及拼装后的对话（`"【场景: " + opponent + "】\n 玩家对策: " + strategy`）发往 Dify 分析。
  - 以十重因果链条（`causal_chain`）展现反制后果。

### 4. 模块四：顶层认知升维（战略家大脑）
- **四大思维自省练习卡片**：穿透历史、穿透结构、穿透因果、穿透自我。点击时滑出相应的核心思考问题。
- **5 层因果推演（Why-Why-Why 练习）**：
  - 用户输入具体职场事件（例如：“副局长试图联合科长架空我”）。
  - 界面强制依次展示 5 个纵深的 Why 输入框：
    - *Why 1*：为什么他们想架空我？
    - *Why 2*：为什么局里存在这种联合架空的动力结构？
    - *Why 3*：为什么该权力机制在此环境下能大行其道？
    - *Why 4*：为什么利益格局没有对此进行制衡约束？
    - *Why 5*：什么是此局盘根错节的最终第一性结构动因？
  - 点击“提交因果穿透”，将 5 层 Why 回答拼接后发送给 Dify API 评估。
  - 展示战略家大脑打分与动力机制解析。
